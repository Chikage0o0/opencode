# 技能设计中的说服力原则

## 概述

LLM 与人类一样会响应相同的说服力原则。理解这种心理学有助于你设计更有效的技能——不是为了操纵，而是为了确保即使在压力下也能遵循关键实践。

**研究基础：** Meincke 等人（2025）使用 N=28,000 次 AI 对话测试了 7 种说服力原则。说服技巧使遵从率提高了一倍以上（33% → 72%，p < .001）。

## 七种原则

### 1. 权威（Authority）
**是什么：** 对专业知识、资质或官方来源的遵从。

**在技能中如何运作：**
- 命令式语言："YOU MUST"、"Never"、"Always"
- 不可协商的框架："No exceptions"
- 消除决策疲劳和合理化借口

**何时使用：**
- 纪律执行类技能（TDD、验证要求）
- 安全关键实践
- 已确立的最佳实践

**示例：**
```markdown
✅ 先写代码再写测试？删掉它。重新开始。没有例外。
❌ 在可行的情况下考虑先写测试。
```

### 2. 承诺（Commitment）
**是什么：** 与先前行为、声明或公开表态保持一致。

**在技能中如何运作：**
- 要求宣布："Announce skill usage"
- 强制明确选择："Choose A, B, or C"
- 使用追踪：TodoWrite 用于清单

**何时使用：**
- 确保技能实际被遵循
- 多步骤流程
- 问责机制

**示例：**
```markdown
✅ 当你找到一个技能时，你必须宣布："I'm using [Skill Name]"
❌ 考虑让你的伙伴知道你在使用哪个技能。
```

### 3. 稀缺性（Scarcity）
**是什么：** 来自时间限制或有限可用性的紧迫感。

**在技能中如何运作：**
- 时间限制要求："Before proceeding"
- 顺序依赖："Immediately after X"
- 防止拖延

**何时使用：**
- 立即验证要求
- 时间敏感的工作流
- 防止"我稍后再做"

**示例：**
```markdown
✅ 完成任务后，在继续之前立即请求代码审查。
❌ 你可以在方便的时候审查代码。
```

### 4. 社会认同（Social Proof）
**是什么：** 遵从他人做法或被认为是正常的行为。

**在技能中如何运作：**
- 通用模式："Every time"、"Always"
- 失败模式："X without Y = failure"
- 建立规范

**何时使用：**
- 记录通用实践
- 警告常见失败
- 强化标准

**示例：**
```markdown
✅ 没有 TodoWrite 追踪的清单 = 步骤被跳过。每次都是如此。
❌ 有些人觉得 TodoWrite 对清单很有帮助。
```

### 5. 团结（Unity）
**是什么：** 共享的身份感、"我们"意识、群体内归属感。

**在技能中如何运作：**
- 协作语言："our codebase"、"we're colleagues"
- 共同目标："we both want quality"

**何时使用：**
- 协作工作流
- 建立团队文化
- 非层级实践

**示例：**
```markdown
✅ 我们是同事，一起工作。我需要你诚实的技术判断。
❌ 如果我错了，你应该告诉我。
```

### 6. 互惠（Reciprocity）
**是什么：** 回报所获利益的责任感。

**如何运作：**
- 谨慎使用——可能感觉被操纵
- 在技能中很少需要

**何时避免：**
- 几乎总是（其他原则更有效）

### 7. 好感（Liking）
**是什么：** 偏好与我们喜欢的人合作。

**如何运作：**
- **不要用于合规**
- 与诚实的反馈文化冲突
- 产生谄媚行为

**何时避免：**
- 纪律执行时永远避免

## 按技能类型的原则组合

| 技能类型 | 使用 | 避免 |
|------------|-----|-------|
| 纪律执行类 | Authority + Commitment + Social Proof | Liking, Reciprocity |
| 指导/技巧类 | Moderate Authority + Unity | Heavy authority |
| 协作类 | Unity + Commitment | Authority, Liking |
| 参考类 | 仅清晰度 | 所有说服技巧 |

## 为什么有效：心理学原理

**明确界线规则减少合理化：**
- "YOU MUST" 消除决策疲劳
- 绝对性语言消除"这是例外吗？"的疑问
- 明确的反合理化对策堵住特定的漏洞

**执行意图创造自动行为：**
- 清晰的触发条件 + 要求的行动 = 自动执行
- "When X, do Y" 比"generally do Y"更有效
- 减少遵从的认知负担

**LLM 是类人的：**
- 训练于包含这些模式的人类文本
- 权威语言在训练数据中先于遵从行为出现
- 承诺序列（声明 → 行动）被频繁建模
- 社会认同模式（每个人都做 X）建立规范

## 道德使用

**正当使用：**
- 确保遵循关键实践
- 创建有效文档
- 防止可预见的失败

**不正当使用：**
- 为个人利益操纵
- 制造虚假紧迫感
- 基于内疚的遵从

**检验标准：** 如果用户完全理解这种技巧，它是否服务于用户的真正利益？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 七种说服原则
- 影响力研究的实证基础

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 使用 N=28,000 次 LLM 对话测试 7 种原则
- 说服技巧使遵从率从 33% 提高到 72%
- Authority、commitment、scarcity 最有效
- 验证了 LLM 的类人行为模型

## 快速参考

设计技能时，问自己：

1. **这是什么类型？**（纪律类 vs. 指导类 vs. 参考类）
2. **我试图改变什么行为？**
3. **哪些原则适用？**（通常纪律类使用 authority + commitment）
4. **我是否组合了太多？**（不要使用全部七种）
5. **这是否道德？**（是否服务于用户的真正利益？）
