Run a `/grilling` session, using the `/domain-modeling` skill.

Goal: clarify requirements only.

Stop once the user's requirements, constraints, acceptance criteria, and open decisions are clear enough to hand off.

Do not write an implementation plan.
Do not propose a step-by-step build plan.
Do not modify files or code.
Do not start implementation.

When requirements are clear, end with a concise requirements summary and explicitly say you are stopping for user confirmation before any planning or code changes.
